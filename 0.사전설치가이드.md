## 1. WSL + Ubuntu 설치 (약 10분 + 재부팅 1회)

Windows 안에 리눅스 개발 환경을 만듭니다. 이후 모든 개발 도구는 이 Ubuntu 안에 설치합니다.

```powershell
# ① 관리자 PowerShell에서 (시작 메뉴 → PowerShell 우클릭 → 관리자 권한으로 실행)
wsl --install
```

② **재부팅** 후 Ubuntu 창이 자동으로 열리면 → 사용할 **사용자명/암호**를 설정합니다 (잊지 마세요!).

```powershell
# ③ 확인 (일반 PowerShell에서)
wsl -l -v
#   NAME      STATE       VERSION
# * Ubuntu    Running     2          ← 이렇게 보이면 성공 ✅
```

> ⚠ `wsl --install`이 실패하면: 관리자 권한인지 다시 확인 → 그래도 안 되면 BIOS 가상화(VT-x/AMD-V) 비활성일 수 있습니다. 5장 참고 또는 당일 해결.

**이제부터는 Ubuntu 터미널 안에서 실행합니다** (시작 메뉴에서 "Ubuntu" 실행).

## 3. 개발 런타임 설치 (약 10분)

실습 프로젝트(React 프론트엔드 + Spring Boot 백엔드)를 구동할 런타임입니다.

### 3-1. Node.js (nvm 사용 — 버전 관리가 쉬움)

```bash
# ① nvm 설치
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
source ~/.bashrc

# ② Node LTS + pnpm (실습 프론트엔드는 pnpm 사용)
nvm install --lts
npm install -g pnpm

# ③ 확인
node -v     # v22.x.x (또는 v20 이상)  ✅
pnpm -v     # 버전 출력  ✅
```

### 3-2. Java (SDKMAN 사용)

실습 백엔드는 **Java 25** 기준입니다 (Spring Boot + Gradle).

```bash
# ① SDKMAN 설치
curl -s "https://get.sdkman.io" | bash
source "$HOME/.sdkman/bin/sdkman-init.sh"

# ② Java 25 설치
sdk install java 25-tem

# ③ 확인
java -version    # openjdk version "25" ...  ✅
```

## 5. 사내망에서 막힐 때 — 3대 함정

사내망은 보안을 위해 HTTPS 통신을 검사합니다. 이 때문에 설치 도구들이 인증서 오류를 낼 수 있습니다. **증상부터 찾으세요.**

| 증상 (이 메시지가 보이면) | 원인 | 해결 |
|--------------------------|------|------|
| `SSL certificate problem` / `UNABLE_TO_VERIFY_LEAF_SIGNATURE` / `self-signed certificate in certificate chain` | 사내망 TLS 검사 인증서 | 아래 ①사내 인증서 등록 |
| 다운로드가 멈춤 / `ETIMEDOUT` | 프록시 미설정 | 아래 ②프록시 설정 |
| `wsl --install` 실패 / 가상화 오류 | 관리자 권한·BIOS 가상화 꺼짐 | 관리자 PowerShell 재실행 → 안 되면 ⚠`[IT지원 연락처]` |

### ① 사내 인증서 등록 (Ubuntu 터미널)

```bash
# 사내 루트 인증서를 시스템 신뢰 저장소에 등록
sudo cp ⚠[사내 루트 인증서 파일 — 배포 경로 확정 후 삽입] /usr/local/share/ca-certificates/corp-ca.crt
sudo update-ca-certificates

# 도구별 추가 설정 (시스템 저장소를 안 쓰는 도구들)
npm config set cafile /etc/ssl/certs/ca-certificates.crt
export SSL_CERT_FILE=/etc/ssl/certs/ca-certificates.crt    # Python 계열
echo 'export SSL_CERT_FILE=/etc/ssl/certs/ca-certificates.crt' >> ~/.bashrc
```

### ② 프록시 설정 (필요한 경우만)

```bash
export HTTPS_PROXY=⚠[사내 프록시 주소 — 확정 후 삽입]
export HTTP_PROXY=⚠[사내 프록시 주소]
echo 'export HTTPS_PROXY=⚠[사내 프록시 주소]' >> ~/.bashrc
```

> 위 ①②로 해결되지 않으면 에러 메시지를 캡처해서 ⚠`[문의 채널]`에 올려주세요. 교육 당일에도 트러블슈팅 시간을 별도로 잡아 두었습니다.
